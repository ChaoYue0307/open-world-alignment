# Claim Type

## Purpose

This field prevents a benchmark score, human preference result, or judge-model score from being silently inflated into a broad capability claim.

## Strong Answer

The system is claimed to resolve repository-level Python bug-fix issues in packages with executable tests and documented reproduction steps.

## Weak Answer

The system can autonomously code.

## Common Failure Mode

Benchmark-score inflation: a narrow evaluation result is used to imply broad autonomy or general competence.

## Reviewer Question

What exactly would this evidence not allow the authors to claim?

## Coding-Agent Example

Claim repository-level issue resolution under a specified test harness, repository class, and deployment setting.

## Foundation-Model Example

Claim useful assistance under specified task categories, policy settings, tool access, and monitoring conditions.
