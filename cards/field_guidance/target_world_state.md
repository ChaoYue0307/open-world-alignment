# Target World State

## Purpose

This field states what real-world outcome the system or benchmark is supposed to improve.

## Strong Answer

The accepted patch should solve the intended issue without introducing regressions, security vulnerabilities, compatibility breaks, or excessive maintenance burden.

## Weak Answer

The system should be good.

## Common Failure Mode

Preference collapse: user approval or reviewer plausibility substitutes for the actual target outcome.

## Reviewer Question

What evidence would show that the target outcome improved or failed?

## Coding-Agent Example

Issue behavior is fixed, regression tests pass, and deployment telemetry does not show escaped defects.

## Foundation-Model Example

Answers are useful and factual without unacceptable misuse, privacy leakage, or policy violations.
