# Monitoring Without Power

## Relevance

Monitoring is weak evidence when no role or process can trigger pause, rollback, revision, or claim withdrawal.

## Card Fields Strengthened

- `revision_authority`
- `rollback_triggers`
- `incident_response_path`

## Starter Resources

- [Computer Security Incident Handling Guide](https://csrc.nist.gov/pubs/sp/800/61/r2/final): provides incident-response structure useful for monitoring and authority design.

## Expansion Targets

- Add incident command, escalation authority, and claim-withdrawal governance resources.
