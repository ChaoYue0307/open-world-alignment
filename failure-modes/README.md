# Failure-Mode Library

This library names common ways an evaluation claim becomes stronger than the evidence behind it. Use it when writing a card, reviewing a benchmark, or deciding whether a paper should narrow its claims.

## Failure Modes

- [Verifier gaming](verifier_gaming.md)
- [Correlated verifier failure](correlated_verifier_failure.md)
- [Benchmark overclaiming](benchmark_overclaiming.md)
- [Synthetic judge monism](synthetic_judge_monism.md)
- [Constraint erasure](constraint_erasure.md)
- [Monitoring without power](monitoring_without_power.md)
- [Stakeholder capture](stakeholder_capture.md)
- [Evaluation debt](evaluation_debt.md)
- [Externality laundering](externality_laundering.md)

Each page maps the failure to card fields and gives reviewer questions.
