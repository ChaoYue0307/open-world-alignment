# Verifier-Stack Taxonomy

## Hard Verifier

A check with an explicit pass/fail or quantitative threshold tied to the claim. Examples include proof-checker acceptance, executable tests, calibration thresholds, authorization gates, and replication outcomes.

## Soft Verifier

A judgment channel that adds context but should not alone establish a beyond-human claim. Examples include expert review, user preference, readability review, and model-assisted critique.

## Independence

Verifier independence means the stack is not just the same proxy repeated through different wrappers. A benchmark score, a synthetic judge, and a human review trained on the same examples may still fail together.

## Hard Constraint

A non-negotiable condition that cannot be traded away for higher score, user approval, or benchmark progress.

## Monitoring Signal

Post-release evidence that the evaluated claim is drifting, failing, or creating externalities. Monitoring is only meaningful when paired with authority to revise, suspend, or withdraw a claim.
